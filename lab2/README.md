# Лабораторная работа №2 — виртуальный датчик для контроля процесса обжига в печи

В репозитории подготовлено решение по заданию из файла «Лабораторная работа №2». Оно включает:
- `lab2_virtual_sensor.ipynb` — Jupyter Notebook с полным пайплайном решения;
- `requirements.txt` — зависимости;
- `README.md` — инструкция по запуску и интерпретации результатов.

## Что делает решение

Ноутбук реализует полный цикл:
1. Загружает минутную телеметрию и лабораторные замеры.
2. Выполняет EDA: пропуски, распределения, частоты, визуализации, анализ задержек.
3. Синхронизирует телеметрию и target с учётом задержки лабораторного измерения 10–15 минут.
4. Строит признаки временных рядов:
   - лаги,
   - скользящие статистики,
   - производные и темпы изменения,
   - календарные признаки.
5. Обучает 3 модели:
   - OLS,
   - Ridge,
   - ExtraTreesRegressor.
6. Считает метрики качества:
   - MAE,
   - RMSE,
   - MAPE,
   - WAPE,
   - Direction Accuracy.
7. Выполняет анализ остатков, permutation importance и SHAP (если пакет доступен).
8. Формирует аналитические выводы после каждого основного пункта.

## Как запустить

### 1. Создать и активировать виртуальное окружение

#### macOS / Linux
```bash

source .venv/bin/activate
```

## Как интерпретировать результаты

### 1. EDA
Смотреть на:
- долю пропусков по каналам телеметрии;
- интервалы между лабораторными замерами;
- динамику признаков и target во времени;
- качество синхронизации по candidate delay 10–15 минут.

### 2. Feature engineering
Ключевая идея решения: целевой показатель зависит не только от текущего состояния печи, но и от её недавней истории. Поэтому в модели включены лаги, rolling mean/std/min/max и признаки изменения сигналов.

### 3. Сравнение моделей
Основная итоговая таблица показывает:
- абсолютную ошибку (`MAE`, `RMSE`),
- относительную ошибку (`MAPE`, `WAPE`),
- точность направления изменения (`Direction_Acc_%`),
- вычислительную эффективность,
- `AIC/BIC` для статистической модели OLS.

### 4. Остатки
В блоке анализа остатков нужно обращать внимание на:
- близость распределения остатков к нормальному;
- наличие/отсутствие автокорреляции;
- наличие признаков гетероскедастичности;
- систематические смещения на графике `остатки vs прогноз`.

### 5. Важность признаков
- `permutation importance` показывает, какие признаки реально улучшают качество модели;
- `SHAP` помогает интерпретировать направление и силу влияния признаков на прогноз.

## Методологические решения

### Почему исключены некоторые признаки
Каналы с почти полной пустотой (`telemetry_12`–`telemetry_15`) исключаются из моделирования, потому что они не дают устойчивого сигнала и в основном добавляют шум.

### Почему пропуски заполняются ffill/bfill
Для минутной телеметрии производственного процесса небольшие локальные разрывы обычно означают технические пропуски записи, а не реальное изменение состояния объекта. Поэтому соседние значения являются разумным приближением.

### Почему не используется PCA
По заданию важна интерпретация физики процесса. PCA уменьшает размерность, но ухудшает технологическую интерпретируемость. Поэтому в этом решении приоритет отдан исходным и инженерным признакам.

### Какие модели признаны неподходящими
В ноутбуке отдельно обосновано, почему для этой задачи не подходят:
- univariate ARIMA / Prophet без телеметрии;
- LSTM/GRU, обучаемые с нуля на небольшом количестве нерегулярных target-замеров;
- kNN-регрессия в условиях высокой размерности и смены режимов.
