# Лабораторная работа №3 — решение

В репозитории подготовлены:

- `lab3_weather_pipeline.ipynb` — Jupyter Notebook с полным пайплайном
- `requirements.txt` — зависимости
- `README.md` — инструкция по запуску и интерпретации результатов

## Что реализовано

### 1) EDA
- проверка структуры и качества данных;
- визуализация временных рядов по каждому городу;
- распределения признаков: histogram, boxplot, Q-Q plot;
- межгородское сравнение температуры и осадков;
- корреляционный анализ;
- проверка стационарности (ADF);
- анализ пропусков и пояснение стратегии обработки.

### 2) Инжиниринг признаков
- календарные признаки;
- циклические признаки `sin/cos`;
- лаги;
- скользящие статистики;
- признаки динамики: скорость изменения температуры, ускорение, темп роста давления;
- климатическая норма и температурная аномалия;
- PCA для визуализации разделимости климатических классов.

### 3) Классификация
Реализована классификация **климатического типа** по 30-дневному окну временного ряда.

Используемые климатические группы:
- `умеренно-континентальный`: Москва, Санкт-Петербург
- `морской субтропический`: Сочи, Геленджик
- `дальневосточный`: Благовещенск, Находка

Модель:
- `RandomForestClassifier` на агрегированных признаках окна.

Метрики:
- Accuracy
- Precision / Recall / F1 (macro и weighted)
- Confusion Matrix
- ROC-AUC (OvR)
- анализ ошибок по месяцам
- permutation importance

### 4) Прогнозирование
Для каждого города формируется отдельный датасет и выбирается лучшая модель по валидации среди:
- `Ridge` (multi-output),
- `RandomForestRegressor`,
- `ExtraTreesRegressor`.

Цель:
- прогноз `temperature_2m` на **30 дней вперёд**.

Метрики:
- MAE
- RMSE
- MAPE
- WAPE
- Direction Accuracy
- анализ остатков: Shapiro-Wilk, Ljung-Box, White test

### 5) Интеграция
Собран единый класс `TwoStageWeatherPipeline`, который:
1. принимает последние 30 дней daily-данных;
2. предсказывает климатический тип;
3. выбирает специализированную модель;
4. строит прогноз температуры на 30 дней.

## Как интерпретировать результаты

### EDA
Смотрите:
- наличие трендов и сезонности;
- межгородские различия;
- распределения температуры и осадков;
- корреляции признаков;
- выводы по стационарности.

### Классификация
Обратите внимание на:
- `Accuracy` и `F1_macro` — основная оценка баланса качества;
- `Confusion Matrix` — какие климатические группы путаются;
- `ROC-AUC OvR` — насколько уверенно модель разделяет классы;
- `Permutation Importance` — какие признаки важнее всего.

### Прогнозирование
Главные показатели:
- `MAE` и `RMSE` — абсолютная и квадратичная ошибка;
- `MAPE` и `WAPE` — относительные ошибки;
- `Direction Accuracy` — насколько хорошо модель угадывает направление изменения температуры.

### Остатки
Если:
- `Ljung-Box p-value` маленький — в остатках осталась автокорреляция;
- `White p-value` маленький — возможна гетероскедастичность;
- `Shapiro p-value` маленький — остатки не нормальны.

Для погодных рядов это нормально; важнее отсутствие сильного смещения и устойчивость качества между городами.

## Практические замечания

1. В notebook прогноз строится в **daily-формате**, так как это наиболее разумно для горизонта 30 дней.
2. Данные изначально почасовые, но для месяца вперёд hourly-модель была бы шумнее и тяжелее.
3. Классификация сделана по климатическим типам, а не по конкретному городу.
4. Этап выбора лучшей модели по валидации позволяет получить более гибкий baseline.
5. Решение можно дальше улучшать через:
   - прогноз осадков,
   - вероятностные интервалы,
   - CatBoost/LightGBM/XGBoost,
   - LSTM/Temporal CNN/Transformer,
   - ансамбли по сезонам.
